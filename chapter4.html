File Path system

Step 1: The First Principle (The Fundamental Truth)


The most fundamental truth is this: A website is not one single file.
                It's a collection of different files (HTML, CSS, JavaScript, images, videos, fonts) that all need to work together. These files are organized into folders, just like documents on your computer.

Step 2: The Core Problem

            Now we have a core problem: If your index.htmfile needs to display an image named ogo.png,, 
            how does the HTML file tell the browser where to findlogo.png?
            ou can't just say src="logo.png" and expect it to work every time. What if the logo is in an imagesfolder? What if it's on a completely different website?

Relative File Paths (The Most Important for Your Projects)

A relative file path gives directions to a file starting from the location of the file you are currently in. You will use this 99% of the time for linking your own files together (images, CSS, other HTML pages).

Linking Files Using Relative Paths

  • Scenario 1: Same Folder
    • Situation: index.html → about.html (both in same folder)
    • Path: Just use the filename.
    • Example:

      •                         <a href ="filename.extension" file name >

  • Scenario 2: Sub-Folder (Going Down)
    • Situation: index.html → images/logo.png (inside a subfolder)
    • Path:Folder name + / + filename.
    • Example

                                <img src ="file pathlocation"alt="describe">

                  eg:  src=images/logo.png

  • Scenario 3: Parent Folder (Going Up)
  • Situation:pages/contact.html → images/logo.png (in a sibling folder)
  • Path:Use ../ to go up one level, then go down into the target folder.
  • Expample

    • < src="../images/logo.png" alt="my website lof >

Absolute File Paths

What It Is

  • An absolute path gives the full, complete address (URL) of a resource.
  • It starts with http:// or https:// when used on the web.
  • It points to the exact location on the internet, not relative to your HTML file.

When to Use

Use absolute paths only for external resources (not on your own website):
  • Linking to another website
  • Loading fonts from Google Fonts
  • Loading images or scripts from other servers

Why Not Use Absolute Path for Local Files?

Do not use full local computer paths like C:/Users/Arjun/Desktop/my-website/images/logo.png because they only work on your computer and break on the internet.

Example of a Wrong Local Absolute Path

<img src="C:/Users/Arjun/Documents/MyPrivatePasswords.txt">

Browsers block this to protect your private files (this protection is called a security sandbox).

Absolute Path in Windows

Format: Drive:\Folder\SubFolder\file.txt

Example: C:\Users\JohnDoe\Documents\report.docx

HTML Boilerplate

      Why We Need It

    1. Browsers need clear instructions before showing any content.
    2. Boilerplate gives essential setup info: HTML version, language, character encoding, page title, and mobile behavior.
    3. Without it, browsers enter quirks mode which can cause unpredictable errors.

    Boilerplate Structure

  • Purpose of Each Part

    • <!DOCTYPE html> Tells browser to use modern HTML5 standards.
    • <html lang="en"> Declares page language (important for SEO & accessibility).
    • <head>Contains setup info (not shown on the page).
    • <meta charset="UTF-8" >Prevents broken characters.
    • <meta name="viewport">Makes site mobile-friendly and responsive.
    • <tiltle>Sets browser tab name.
    • <body>Holds all visible content.

    Why Not Skip It

  • Prevents quirks mode
  • Prevents broken characters
  • Enables responsive design
  • Improves SEO and accessibility

Boilerplate = foundation of a stable, professional website.

    Multipage Websites
